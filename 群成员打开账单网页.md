# 群成员打开「查看本群账单」网页

## 原因

- Bot 显示 **「本地模式」** 是正常的（Polling 收消息）。
- 账单链接写在 `.env` 的 `WEB_BASE_URL=http://127.0.0.1:8081`。
- **127.0.0.1 只能在你自己的电脑上打开**，群里其他人（柬埔寨用手机）点链接 **一定打不开**。
- **Neon 云数据库** 只存数据，没有给群成员点的网页；网页要靠 **web.py + 公网地址**。

## 你要同时运行两个程序

| PyCharm 配置 | 作用 |
|-------------|------|
| **运行 Bot** | `app.py`，Telegram 记账 |
| **运行 Web后台** | `web.py`，网页账单（端口 8081） |

## 让群成员能打开（ngrok，约 5 分钟）

### 1. 安装 ngrok

https://ngrok.com/download  
注册后执行（按官网说明配置 authtoken）：

```bash
ngrok config add-authtoken 你的token
```

### 2. 启动 Web 后台

PyCharm → **运行 Web后台**（保持运行）

### 3. 新开终端，暴露 8081

```bash
ngrok http 8081
```

复制 **Forwarding** 里的 **https** 地址，例如：

```text
https://abcd-1234.ngrok-free.app
```

### 4. 改 `.env`

增加或修改：

```env
PUBLIC_WEB_BASE_URL=https://abcd-1234.ngrok-free.app
```

（不要带末尾斜杠）

### 5. 重启 Bot

PyCharm → 停止再 **运行 Bot**。

群里再发账单，按钮会变成可打开的 **https** 链接（群成员手机也能进）。

## 仅自己电脑测试（群成员仍打不开）

浏览器打开（需先运行 Web后台）：

```text
http://127.0.0.1:8081/
```

用 `.env` 里的 `WEB_TOKEN` 登录。

## 以后正式使用

把 Bot + web.py 部署到有域名的服务器（Render、VPS 等），把 `PUBLIC_WEB_BASE_URL` 设成 `https://你的域名`。
