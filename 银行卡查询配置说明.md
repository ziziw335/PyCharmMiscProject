# 银行卡：开户行 vs 姓名/手机号

## 两种能力区别

| 能力 | 输入 | 输出 | 是否合规常见 |
|------|------|------|--------------|
| **查卡 / BIN** | 只要卡号 | 开户行、卡类型 | ✅ 已内置 |
| **四要素核验** | 姓名 + 身份证 + 卡号 + 手机 | 四项是否一致 | ✅ 需买 API |
| **反查姓名手机** | 只要卡号 | 姓名、手机号 | ❌ 银行不公开 |

Bot 里提示的「无法仅凭卡号查姓名/手机」是指：**没有合法公开接口**把卡号反查成持卡人隐私。

---

## 方案一：只查开户行（已完成，免费）

私聊发送：

- `银行卡查询` 或 `查卡 6222021234567890123`

使用本地 `data/bank_bins.json`，无需配置。

---

## 方案二：四要素核验（查「是否同一人」，推荐）

### 1. 购买接口

任选其一（需企业/个人实名、按次计费）：

- [易源 - 银行卡四要素](https://www.showapi.com/apiGateway/view/1072/5)
- [阿里云云市场 - 银行卡四要素](https://market.aliyun.com/apimarket/detail/cmapi00038220)
- [阿里云官方 BankMetaVerify](https://help.aliyun.com/zh/id-verification/information-verification/developer-reference/bank-card-verification)（需 SDK，配置较复杂）

购买后得到 **APPCODE**（易源）或 **AppKey/Secret**（阿里云）。

### 2. 配置 `.env`

```env
# 易源示例
BANK_VERIFY_APPCODE=你从控制台复制的APPCODE
BANK_VERIFY_API_URL=https://ali-bankcard4.showapi.com/bank4
```

若供应商网关地址不同，只改 `BANK_VERIFY_API_URL`。

### 3. 重启 Bot

```bash
systemctl restart telegram-bot
```

### 4. 在 Telegram 使用

发送 **「四要素核验」**，再按提示发送四行：

```
张三
110101199001011234
6222021234567890123
13800138000
```

返回 **一致 / 不一致**（不会把别人的姓名手机号「查出来」，只验证你提供的四项是否匹配）。

## 方案三：BANK_CARD_API_URL（仅当供应商明确支持）

若你购买的「银行卡查询」接口文档写明返回 `name`、`phone` 字段，可配置：

```env
BANK_CARD_API_URL=https://接口地址?cardNo={card}
BANK_CARD_API_KEY=密钥
```

Bot 会在「查卡」时合并显示这些字段。**大多数归属地接口不返回姓名手机**，买前请看文档。

---

## 安全与合规

- 仅用于**本人授权**或**合法业务**核验，勿批量爬取他人信息。
- 勿使用来路不明的「卡号查姓名」社工库接口（违法风险）。
- APPCODE、密钥勿提交 Git、勿发到群里。
